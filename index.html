GitHub Governance and Accreditation for the Met Office

Welcome to the Met Office GitHub Pages

Please take time to read the following important guidance before committing any code to the Met Office GitHub:

Before using either public or private repositories at https://github.com/MetOffice, you must ensure that your Information Assurance Officer (IAO) has agreed that it is acceptable to host your source code on GitHub and that there is no risk to the reputation or operational effectiveness of the Met Office through loss of this intellectual property

  • You should ensure that you maintain a record of this agreement with your IAO - an email conversation is acceptable
  • You confirm that anything you post at https://github.com/MetOffice is classed as below Official â€“ Sensitive
  • Choose the appropriate location for your source code on https://github.com/MetOffice: remember that anything placed in a public repository will be available to any GitHub user and that by default you will have accepted GitHub's Terms of Service which allow those users to view and fork your repository

GitHub strongly encourage users to include an open source license agreement with any source code posted on GitHub . Please look at the following page for how to include this on a GitHub repository and here for guidance on the appropriate open source licensing agreement.

A list of IAOs at the Met Office is maintained by the TIS Programme Office. For any further guidance please contact a member of the CM Team within TIS.
